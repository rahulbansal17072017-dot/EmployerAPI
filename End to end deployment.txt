Step 1: Update GitHub Secrets

You already have:

✅ EC2_HOST
✅ EC2_USER
✅ EC2_KEY

Now add:

DOCKER_USERNAME
DOCKER_PASSWORD

For DOCKER_PASSWORD, use either:

your Docker Hub password, or
preferably a Docker Hub Personal Access Token.

Step 3: Remove the old deployment

Delete these steps from your workflow:

dotnet publish

↓

scp publish/*

↓

systemctl restart employerapi

They are no longer needed because Docker manages the application.

Step 4: Add Docker Login
- name: Login to Docker Hub
  run: |
    echo "${{ secrets.DOCKER_PASSWORD }}" | docker login \
      -u "${{ secrets.DOCKER_USERNAME }}" \
      --password-stdin
Step 5: Build the image
- name: Build Docker Image
  run: |
    docker build -t ${{ secrets.DOCKER_USERNAME }}/employerapi:latest .
Step 6: Push the image
- name: Push Docker Image
  run: |
    docker push ${{ secrets.DOCKER_USERNAME }}/employerapi:latest
Step 7: Deploy on EC2

Replace the old systemctl restart step with:

docker pull username/employerapi:latest

docker stop employerapi || true

docker rm employerapi || true

docker run -d \
-p 5000:8080 \
--name employerapi \
username/employerapi:latest
Step 8: Test

My recommendation

Before changing your workflow, save a copy of your current .github/workflows/deploy.yml. It worked for the non-Docker deployment, so it's a good backup. Then modify it to use Docker. If something goes wrong, you'll have a known working version to compare against.
